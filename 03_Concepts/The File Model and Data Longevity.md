# **The File Model and Data Longevity**

Obsidian operates directly on a local file system directory, referred to as a vault.3 Because it does not rely on a proprietary backend relational database or a locked-in proprietary schema, the data maintains maximum portability and longevity.2 Files can be read, edited, indexed, and mutated by any external script, integrated development environment, or command-line tool without requiring API access or platform permission. This architectural choice inherently mitigates the risk of platform decay, corporate acquisition, or vendor lock-in. The longevity of plain text ensures that even if Obsidian as an application ceases to exist, the underlying knowledge graph remains entirely intact and accessible.

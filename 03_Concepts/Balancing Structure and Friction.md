# **Balancing Structure and Friction**

The primary risk in schema design is over-structuring. If a user is required to manually populate twenty distinct YAML fields to create a single note, the friction will inevitably halt system usage and lead to abandonment.42 Frontmatter must be ruthlessly prioritized, limited to core properties that drive specific queries, automations, or retrieval filters.

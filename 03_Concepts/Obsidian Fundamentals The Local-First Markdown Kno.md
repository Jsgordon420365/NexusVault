# **Obsidian Fundamentals: The Local-First Markdown Knowledge Substrate**

Obsidian’s architecture is deliberately contrarian in an era dominated by cloud-native, proprietary software-as-a-service platforms. At its core, Obsidian operates on a foundational philosophy: data must be local-first, stored in plain text utilizing the Markdown format, and connected via first-class bidirectional links.1 This design philosophy ensures that the system serves as an enduring infrastructure rather than an ephemeral application.

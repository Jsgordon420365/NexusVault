# **The Mechanics of Persistent Memory**

Interactions with frontier models are inherently stateless; each session begins with a blank context window. A persistent memory mesh solves this localized amnesia by providing a three-layered architecture.21 The first layer is working memory, consisting of raw, timestamped session logs that record the exact dialogue between the human and the machine.21 The second layer is distilled memory, representing state files that summarize the current status of a project, active tasks, and recent architectural decisions.16 The final layer is canonical memory, comprising the durable, highly polished evergreen notes that represent established facts and synthesized concepts within the knowledge graph.21

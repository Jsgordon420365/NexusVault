# **Extensibility and the Plugin Ecosystem**

Obsidian’s core application is intentionally minimalist, relying on an extensive, community-driven plugin ecosystem to extend its functionality.1 Plugins enable advanced Dataview queries, Kanban boards, artificial intelligence chat integrations, and direct Git synchronization.7 However, this extensibility introduces significant architectural complexity. Heavily modified vaults can become brittle if plugins overwrite core Markdown standards or introduce proprietary syntax that external parsers and language models cannot reliably interpret.
